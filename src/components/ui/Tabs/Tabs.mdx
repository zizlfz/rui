# Tabs

Tabs organize content into separate views, allowing users to switch between them while staying in the same context.

## Installation

```bash
rui add tabs
```

## Usage

```tsx
import { Tabs, TabList, Tab, TabPanels, TabPanel } from '@/components/ui/Tabs';

<Tabs>
  <TabList>
    <Tab id="overview">Overview</Tab>
    <Tab id="details">Details</Tab>
    <Tab id="settings">Settings</Tab>
  </TabList>
  <TabPanels>
    <TabPanel id="overview">Overview content</TabPanel>
    <TabPanel id="details">Details content</TabPanel>
    <TabPanel id="settings">Settings content</TabPanel>
  </TabPanels>
</Tabs>
```

## Sub-components

`Tabs` is composed of five sub-components: `Tabs`, `TabList`, `Tab`, `TabPanels`, and `TabPanel`. Each tab in the `TabList` corresponds to a `TabPanel` with the same `id`.

<DocDemo>
  <Tabs>
    <TabList>
      <Tab id="overview">Overview</Tab>
      <Tab id="details">Details</Tab>
      <Tab id="settings">Settings</Tab>
    </TabList>

      <TabPanel id="overview">This is the overview panel with general information.</TabPanel>
      <TabPanel id="details">This is the details panel with specific content.</TabPanel>
      <TabPanel id="settings">This is the settings panel with configuration options.</TabPanel>
    </TabPanels>
  </Tabs>
</DocDemo>

## Default Selected Tab

Use `defaultSelectedKey` to set which tab is selected by default. When not provided, the first tab is selected.

```tsx
<Tabs defaultSelectedKey="details">
  <TabList>
    <Tab id="overview">Overview</Tab>
    <Tab id="details">Details</Tab>
  </TabList>
  <TabPanels>
    <TabPanel id="overview">Overview content</TabPanel>
    <TabPanel id="details">Details content</TabPanel>
  </TabPanels>
</Tabs>
```

## Controlled Selection

Use `selectedKey` and `onSelectionChange` to control the selected tab programmatically.

<DocDemo>
  {(props) => (
    <Tabs {...props}>
      <TabList>
        <Tab id="overview">Overview</Tab>
        <Tab id="details">Details</Tab>
      </TabList>
      <TabPanels>
        <TabPanel id="overview">Overview content</TabPanel>
        <TabPanel id="details">Details content</TabPanel>
      </TabPanels>
    </Tabs>
  )}
</DocDemo>

## Disabled Tabs

Use `isDisabled` on a `Tab` to prevent selection. Disabled tabs are visible but not interactive.

<DocDemo>
  <Tabs>
    <TabList>
      <Tab id="overview">Overview</Tab>
      <Tab id="details">Details</Tab>
      <Tab id="settings" isDisabled>Settings</Tab>
    </TabList>
    <TabPanels>
      <TabPanel id="overview">Overview content</TabPanel>
      <TabPanel id="details">Details content</TabPanel>
      <TabPanel id="settings">Settings content</TabPanel>
    </TabPanels>
  </Tabs>
</DocDemo>

## Keyboard Interactions

| Key | Description |
| --- | --- |
| `Tab` | Moves focus to the selected tab. |
| `ArrowLeft` / `ArrowRight` | Moves focus between tabs in horizontal orientation. |
| `ArrowUp` / `ArrowDown` | Moves focus between tabs in vertical orientation. |
| `Home` | Moves focus to the first tab. |
| `End` | Moves focus to the last tab. |
| `Enter` / `Space` | Activates the focused tab. |

## Accessibility

Tabs follow the WAI-ARIA Tabs Pattern. The `TabList` has `role="tablist"`, each `Tab` has `role="tab"`, and each `TabPanel` has `role="tabpanel"`. ARIA attributes like `aria-selected`, `aria-controls`, and `aria-labelledby` are managed automatically by React Aria Components.
